# Asset Licensing Policy

This project helps record licensing evidence; it does not provide legal advice or guarantee that a particular asset is safe for a particular use.

## Default policy

- Prefer CC0 and clearly identified public-domain assets.
- Allow CC BY when complete attribution metadata is available.
- Warn on CC BY-SA and require user confirmation.
- Reject NC licenses for commercial-capable projects by default.
- Reject ND licenses for edited video workflows by default.
- Reject editorial-only and unknown-rights assets from the default candidate pool.
- Treat platform-specific free licenses as distinct policies, not as “open source.”

Copyright permission does not automatically resolve trademarks, privacy, publicity/personality rights, moral rights, or third-party elements depicted in an asset.

## Required evidence

Every selected remote asset must retain its creator, original source page, provider, license or rights statement, attribution requirement, provider terms URL, retrieval time, and a minimal rights snapshot. Users must be able to return to the original page before publishing.

The source and license catalogs live under `catalog/`. Entries are added in M1 and must cite official documentation with a last-verification date.
